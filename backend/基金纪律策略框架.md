# 基金纪律策略框架

> 每只基金套进模板填参数即可执行。框架本身不需要改动。
> 核心原则：**触发用零滞后指标，调节用滞后指标。**
> **所有信号都是提示，最终人工执行。**

---

## 一、核心原则

- **触发用零滞后指标，调节用滞后指标**：触发信号要敏感（怕错过），仓位信号要稳健（怕被骗线）。两层分离，避免"一个指标定生死"。
- **所有信号都是提示，人工执行**：框架只负责算信号、算额度、卡硬约束、记录。所有操作最终人工拍板，可 override（需记录原因）。
- **顺势思路**：建仓顺势进，移动止盈顺势出，加仓用回撤做左侧摊薄。
- **不看估值**：当前赚钱的强势基金估值普遍偏高，用估值当禁买闸会把该买的品种挡在外面。本框架只用技术面调节。

> 说明：严格意义上"零滞后"不存在，回撤幅度是基于已发生价格的反应型指标，只是没有均线类平滑延迟。本框架沿用这个表述，心里清楚即可。

---

## 二、基准锚

**相对前高净值回撤**：

```
回撤 = (前高净值 - 当前净值) / 前高净值
```

前高定义：该基金历史最高累计净值（含分红再投的复权净值）。

---

## 三、场外基金通用硬约束（所有基金适用，不可调）

1. **净值确认延迟**：T+1 甚至 T+2 确认。当天看到的回撤，实际成交在第二天净值。触发信号发出后，15:00 前下单。
2. **赎回到账延迟**：通常 2–7 天。急需用钱的仓位不能全压场外基金。
3. **短期持有惩罚费**：持有不满 7 天赎回收 1.5% 惩罚费。建仓后 7 天内不得赎回（除非逻辑破坏止损）。
4. **大额申购限制**：部分热门基金限购，加仓前确认是否限额。
5. **天然滞后容差**：零滞后触发在实操上有 1–2 天天然滞后，档位设计要留缓冲，不要卡在精确点位。

---

## 四、基金分类与默认回撤档位

不同类型基金波动率差异大，回撤档位不能一刀切。下面是默认值，每只基金可微调。

### 默认加仓档位

| 档位      | 宽基   | 行业   | 主动   | 混合   |
|---------|------|------|------|------|
| 一档（轻仓加） | -8%  | -15% | -12% | -12% |
| 二档（标准加） | -15% | -25% | -20% | -22% |
| 三档（重仓加） | -25% | -35% | -30% | -32% |
| 四档（极限加） | -35% | -45% | -40% | -40% |

> 原则：档位越深，加仓额越大（金字塔加仓），但受单品种仓位上限硬约束。

### 为什么这样分

- **宽基**（沪深300、中证500等）：回撤 30% 已是大级别熊市，档位收紧。
- **行业**（半导体、医药、新能源等）：回撤 30% 是常态，45% 也不罕见，档位拉宽。
- **主动**：取决于基金经理风格，介于宽基和行业之间。
- **混合**：权益仓位灵活，按偏股混合估，档位接近主动。

## 五、建仓（首次买入）

建仓和加仓是两种性质不同的决策：建仓是"选品+进入"，加仓是"仓位管理"。分开处理。

### 建仓触发条件（趋势确认，写死不参数化）

满足以下**全部**条件，提示建仓：

- 价格在年线上方 且 年线向上
- 且 近 60 日创过阶段新高

> 这是顺势建仓，不是等回调抄底。跟移动止盈的顺势思路对齐。

### 建仓首笔额度

```
建仓首笔 = 计划总仓位 × 10%
```

- **计划总仓位**：用户建仓时设定，该基金目标投入总额（输入数据，非参数）
- **固定 10%，不参数化**：试探性首笔，剩余 90% 留给四档加仓
- 四档基础加仓额之和应 ≤ 计划总仓位 × 90%

### 建仓流程

满足触发条件 → 提示建仓 → 人工确认 → 买入首笔（计划总仓位 × 10%）→ 标记"已建仓" → 进入加仓框架

> 建仓选品最终人工判断。趋势条件只是机械参考，赛道逻辑、经理能力等基本面由人决定。可手动标"已建仓"直接进加仓框架（override，记录原因）。

---

## 六、加仓（已有仓位追加）

### 机制：方案B——每档只加一次 + 反弹清空

**档位区划分**（以宽基为例）：

| 回撤区间      | 档位   |
|-----------|------|
| < 8%      | 未触发区 |
| 8% ~ 15%  | 一档   |
| 15% ~ 25% | 二档   |
| 25% ~ 35% | 三档   |
| ≥ 35%     | 四档   |

**触发**：回撤加深进入新档，且该档未加 → 提示加仓，标记该档已加（记录时间戳）。

**清空**：回撤变浅换到更浅档 → 清空所有比当前档更深的已加标记。

- 从二档反弹到一档区 → 清空二、三、四档标记
- 反弹到未触发区（< 一档阈值）→ 清空全部标记，下一轮从一档重新开始

**举例**：跌到 -15% 加二档 → 跌到 -25% 加三档 → 反弹到 -12%（回到一档区）→ 清空二、三档 → 再跌到 -15% → 重新加二档。

### 强制冷静

单周跌幅 > 阈值 → 提示暂停加仓，等周线企稳。这是强提示，可 override（记录原因）。

> 跟"破位观望"侧重不同：强制冷静看跌幅速度，破位观望看趋势+量，两者并存。

---

## 七、调节层（决定加多少）

触发后，用滞后指标给仓位打系数。不是 0/1 开关，而是乘数。

### 调节指标与系数

| 调节指标           | 看什么          | 系数  |
|----------------|--------------|-----|
| **年线（250日均线）** | 价格在年线上方且年线向上 | 1.0 |
|                | 价格在年线上方但年线向下 | 0.7 |
|                | 价格在年线下方且年线向下 | 0.4 |
| **周MACD**      | 底背离          | 1.2 |
|                | 绿柱缩小（接近金叉）   | 1.0 |
|                | 红柱但缩小        | 0.9 |
|                | 绿柱扩大         | 0.6 |
| **成交量**        | 地量企稳（缩量横盘）   | 1.2 |
|                | 正常           | 1.0 |
|                | 放量下跌         | 0.5 |

> 主动基金看其持仓主指数或基金净值本身的年线/MACD/成交量。

### 系数合成规则

```
最终系数 = 年线系数 × 周MACD系数 × 成交量系数
最终系数 = clamp(最终系数, 0.3, 1.5)   # 防止极端值
实际加仓额 = 该档基础加仓额 × 最终系数
```

> 最低 0.3 意味着技术面全面走弱时仍加 30% 基础额——由"破位观望规则"和硬约束兜底。

### 破位观望规则（强提示，可 override）

当**同时**满足以下两项，提示不建议加仓：

- 年线状态 = 价格在年线下方且年线向下（系数 0.4）
- 成交量状态 = 放量下跌（系数 0.5）

恢复条件：周MACD 出现底背离，或绿柱开始缩小。任一满足即解除观望。

> 这条只看技术面，不看估值。逻辑是趋势破位 + 放量出逃时，等恐慌释放完再加，不接飞刀。强提示，人工可 override（记录原因）。

## 八、卖出

| 类型     | 触发                             | 操作          | 锚     |
|--------|--------------------------------|-------------|-------|
| 移动止盈   | 从持有期最高净值回落 X%                  | 减 1/3 或 1/2 | 持有期高点 |
| 逻辑破坏止损 | 行业赛道逻辑崩塌 / 主动基金经理换人 / 指数编制重大变更 | 全清，不卡价位     | 不限    |
| 再平衡减仓  | 单只占比超过预设上限                     | 减到上限内       | 持仓结构  |

**优先级**：逻辑破坏止损 > 移动止盈 > 再平衡。

> 去掉了目标止盈（固定盈利目标会卖飞强势品种）和估值极端止盈（不看估值）。
> 移动止盈用"持有期最高净值"，不是"前高净值"——前者算我持有后的回吐，后者算市场回撤，两个高点不同。

---

## 九、硬约束（真硬约束，不可 override）

1. **单品种仓位上限**：单只基金不超过总仓位的 20%（行业基金 15%）。
2. **单类型仓位上限**：单一行业类型不超过总仓位的 30%。
3. **总仓位上限**：权益类基金不超过可投资金的 80%，永远留 20% 现金/货基。
4. **单次加仓上限**：单次加仓额不超过该品种剩余可用仓位的 50%。
5. **7 天内不赎回**：除非逻辑破坏止损，否则建仓后 7 天内不卖。
6. **不借钱加仓**：只用闲钱，不用杠杆。

## 十三、执行清单（每次操作前过一遍）

### 建仓前

1. ☐ 检查建仓触发：年线向上 + 60 日新高
2. ☐ 人工确认选品（赛道逻辑、经理、基本面）
3. ☐ 设定计划总仓位
4. ☐ 计算建仓首笔 = 计划总仓位 × 10%
5. ☐ 过硬约束（单品种上限、总仓位 80%）
6. ☐ 15:00 前下单，标记"已建仓"

### 加仓前

1. ☐ 确认基金状态 = 已建仓
2. ☐ 计算当前相对前高回撤
3. ☐ 判断当前回撤档位
4. ☐ 检查该档是否已加（方案B状态）
5. ☐ 检查单周跌幅是否触发强制冷静
6. ☐ 查三个调节指标系数，算最终系数
7. ☐ 检查破位观望（年线下方向下 + 放量下跌 → 强提示不加）
8. ☐ 计算实际加仓额 = 基础额 × 最终系数
9. ☐ 过硬约束五条
10. ☐ 人工确认，15:00 前下单，更新档位状态

### 卖出前

1. ☐ 检查是否逻辑破坏（优先级最高）
2. ☐ 计算从持有期最高点回落幅度
3. ☐ 检查单只占比是否超上限
4. ☐ 确认持有满 7 天（否则等，除非逻辑止损）
5. ☐ 按优先级决定卖出类型和比例
6. ☐ 人工确认，15:00 前下单，记录操作

## 附：框架使用说明

1. 每只新基金建仓前，先查它的历史最大回撤分布，校准默认档位是否合理。
2. 每月更新一次各基金的"前高净值"。
3. 每次操作后填操作记录，季度复盘一次。
4. 框架是死的，纪律是活的——一旦定好参数，机械执行，不要临场改阈值。
5. 所有信号都是提示，最终人工拍板；override 时记录原因。
