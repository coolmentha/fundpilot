# passed 判定：去 all-in 收益门槛，回撤标尺从 all-in 换 DCA

> **Status: superseded by [ADR-0010](./0010-passed-criteria-calmar-vs-dca.md)** —— 本 ADR 的回撤标尺（策略回撤 ≤ dca 回撤）
> 在实战数据（基金 1958）上暴露问题：绝对回撤约束惩罚用合理风险换合理收益的策略，故回撤维度改为 Calmar 比较。本 ADR 保留作为决策演进留痕。

`BenchmarkCalculator.judgePassed` 原要求策略收益**严格大于三条基准**（hs300 / all-in / dca）**且**回撤 ≤ all-in 回撤。
本次变更：**收益门槛去掉 all-in**（只比 hs300 / dca），**回撤标尺从 all-in 换成 dca**。all-in 指标仍计算并落库展示基金自然回撤，
但不再参与 passed 判定。判定口径全链路统一——calibrate 主路径与寻优 test 集共用同一个 `judgePassed`。

## Considered Options

- **A. 去 all-in 收益门槛 + 回撤标尺换 dca（已采纳）**：收益须 > hs300 ∧ > dca；回撤须 ≤ dca 回撤。all-in 退出判定，仅展示。
- **B. 只去 all-in 收益门槛，留 all-in 回撤约束**：收益须 > hs300 ∧ > dca；回撤须 ≤ all-in 回撤。all-in 降级为纯回撤标尺。
- **C. all-in 完全移除（收益 + 回撤都去）**：收益须 > hs300 ∧ > dca；无回撤约束。
- **D. 回撤标尺改挂 hs300**：收益须 > hs300 ∧ > dca；回撤须 ≤ hs300 回撤。

## Consequences

选 A 的核心理由：**每条基准挂到它领域语义最匹配的维度上**。

1. **收益门槛去 all-in**——all-in 是"期初一次满仓"，在单边涨的短窗口（如寻优 test 段 0.3 年）**不可战胜**：
   满仓吃涨幅、零摩擦，任何分批建仓的纪律策略都跑不赢。用它卡收益等于惩罚纪律策略的弱项。
   更关键的是，CONTEXT.md「风险调整收益」明确把"纯收益率最大化"列为 _Avoid_（"可能选出回撤极大的激进参数"）——
   all-in 收益正是那个激进参数的化身。让纪律策略必赢 all-in，与框架规避激进参数的精神相悖。
   dca 才是与纪律策略同语义的对照（都分批建仓、不择时满仓），策略跑不赢最朴素的定投才算择档加仓没带来 alpha。
2. **回撤标尺换 dca**——all-in 回撤是该基金净值序列的自然最大回撤（满仓市值正比净值），
   "策略回撤 ≤ all-in 回撤"看似成立实则**约束形同虚设**：分批建仓本就该降回撤，满仓回撤本就最大，这条门槛永远容易过。
   dca 已通过分散入场降低回撤，"策略回撤 ≤ dca 回撤"才有约束力——策略回撤超过定投，说明择档加仓在**制造**风险而非控制风险。
   这是比 all-in 更严、领域语义更准的回撤闸门。
3. **不选 C**——完全去掉回撤约束会让回撤 50% 但收益侥幸赢基准的参数过关，违背「风险调整收益」规避激进参数的 _Avoid_。
4. **不选 D**——hs300 回撤与基金自身波动脱钩：一只高波动行业基金回撤天然 > hs300，用 hs300 卡回撤不公平。
   dca 是同基金同语义对照，无此问题。

## 代价

判定口径变更后，**历史回测记录的 passed 值不重算**——它们是历史留痕。
原本因"输 all-in 收益"判 false 的记录，按新标准本该 true；原本因"回撤 ≤ all-in"判 true 的记录，
若回撤 > dca 回撤，按新标准本该 false。这些历史值保持不动，作为历史事实；新判定只影响今后跑的回测。
`judgePassed` 签名保留 allIn 参数（调用方仍计算并落库展示），内部不再使用——这是为维持调用方契约、避免无领域收益的签名扩散改动，
未来若调用方重构可一并清理。
