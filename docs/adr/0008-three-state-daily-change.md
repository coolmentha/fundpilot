# 三态今日涨跌模型

「今日涨跌」定义为随交易时段演进的三态概念——盘前 = 0、盘中 = fundgz 实时估值、盘后 = 当日落库净值算。
判定靠"当前时间 + 当日净值是否已落库"（非纯时间），今日盈亏/总盈亏随之三态，用"市值×涨跌幅"统一口径。
推翻旧定义"今日涨跌 = T-1 vs T-2"（那实际是昨日涨跌）。

## Considered Options

- **A. 三态单字段 + 时间+落库判定（已采纳）**：「今日涨跌」是单一概念，值随时段切换数据源。盘前<9:30=0；
  盘中（≥9:30 且当日净值未落库）= fundgz gszzl，isEstimated=true；盘后（当日净值已落库）= 落库累计净值算。
  15:00-20:00 待公布时段显示最后一次估值。今日盈亏 = 昨日市值×今日涨跌幅，总盈亏盘中 = 昨日总盈亏×(1+涨跌幅)。
  口径统一不引入单位净值 gsz。当晚 20-23 点每分钟确认当日净值落库，让盘后实际值生效。
- **B. 两字段并存（dailyChangePct + estimatedChangePct）**：盘中估算与盘后实际作为两个独立字段，前端自行选择展示。
  代价：前端逻辑复杂（何时显哪个），且两字段口径/时点不同易混淆，"今日涨跌"语义被稀释。
- **C. 纯时间判定**：9:30-15:00 调 fundgz，其余时段用落库净值。简单但盘后净值未公布时（15:00-20:00）会显示错误的 0 或 T-1 旧值。

## Consequences

选 A 的核心理由：**单一概念 + 时间+落库判定 + 口径统一**。

1. 单一概念——「今日涨跌」是用户视角的"今天的变化"，不应因数据源不同拆成两字段。三态是同一概念在不同时段的值，
   isEstimated 标记区分估算/实际，前端只需加"估"标记，无需切换字段逻辑。
2. 时间+落库判定——盘后净值公布时间因基金而异（19:00-22:00 不等），纯时间判定会在 15:00-20:00 显示错误值。
   "当日净值是否已落库"是真实状态，判定准确。代价是需当晚确认拉取机制（DailyNavConfirmJob）支撑落库判定。
3. 口径统一——今日盈亏 = 市值×涨跌幅、总盈亏 = 昨日总盈亏×(1+涨跌幅)，都用涨跌幅比例推算，不引入单位净值 gsz。
   涨跌幅是比例（同日不除权时单位/累计净值涨跌幅一致），无口径问题；绝对盈亏若用 gsz 会对分红基金失真，故用比例规避。

不选 B：两字段并存让"今日涨跌"语义分裂，前端需判断何时显哪个，且估算/实际口径不同（单位 vs 累计净值）易混淆。
不选 C：纯时间判定在 15:00-20:00（净值未公布）显示错误值，破坏"今日涨跌"准确性。

## 代价

- 需当晚确认拉取机制（DailyNavConfirmJob 20-23 点每分钟）支撑盘后落库判定，增加一条定时任务。
- 估算盈亏是近似（市值×涨跌幅），非精确的份额×净值变动；对未分红基金两者数学等价，分红基金有微小差异（可接受）。
- 列表按需拉 fundgz：N 只基金 N 次请求，依赖限流（issue #35 Bucket4j）；本期基金数量少可接受，不做跨实例缓存。
