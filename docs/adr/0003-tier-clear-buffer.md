# 反弹清空判定加 0.5% 缓冲带，偏离框架原文

`backend/基金纪律策略框架.md §六` 明确"反弹清空"按精确档位边界判定（举例："反弹到 -12%（回到一档区，因为
-12% ∈ [-15%, -8%]）→ 清空二、三、四档"）。本期实现选择**只在清空判定一侧**加 0.5% 缓冲带——`drawdown < tierNDrawdown - 0.005`
才清空 N 档。

## Considered Options

- **Z1. 严格按框架原文（精确边界清空）**：每天按当前回撤所在档位识别"更深档"清空。简单忠实于框架。
- **Z2-a. 只缓冲清空侧 0.5%（已采纳）**：加档判定不变（精确触发），清空判定收紧 0.5% 才触发。
- **Z2-b. 只缓冲加档侧 0.5%**：加档要超阈值 0.5% 才触发，清空按原文。违背"越跌越买"精神。
- **Z2-c. 两侧对称缓冲**：进档要超阈值 0.5%，出档要回阈值 0.5%。形成 1% "档位锁定区"，最稳定但延迟加档。

## Consequences

选 Z2-a 的核心理由：**抖动的真实成本不对称**。"加 - 清 - 加" 的损失主要在"清错了"
——清空标记后下一次跌到该档又触发加仓信号，可能让用户在同一档"买两次"，超出该档计划比例。"该加没加"
的损失是机会成本（净值再涨就错过），但移动止盈和深档加仓会兜底。所以缓冲应优先保护"不要错误清空"。

0.5% 的缓冲强度选择依据：四档回撤阈值间距最近 7%（一档 -8% 到二档 -15%），0.5% ≈ 7% 的 1/14，覆盖正常日间波动但不会把档位边界整体推深太多。

落地为 `support/HardConstraintConfig.TIER_CLEAR_BUFFER = 0.005` 常量，不做成可调参数。

## 偏离框架原文的合理性

框架§附 第 4 条："框架是死的，纪律是活的——一旦定好参数，机械执行，不要临场改阈值。" 严格读这条原则就该选 Z1。Z2-a
的偏离需要承担"违纪"的合理性责任：

1. 框架的回撤档位假设市场净值变化"连续平滑"，但实际是离散日频，边界震荡是真实场景。
2. 缓冲带不改变档位本身，只改变"识别脱离档位"的灵敏度——纪律语义保留，工程鲁棒性增强。
3. 选择不可调参数（写死 0.5%）维持"机械执行不临场改阈值"的纪律精神——这是参数而非策略意图。

如果实战发现 0.5% 缓冲不足以阻止抖动或反而延迟有效信号，调整 `TIER_CLEAR_BUFFER` 常量即可；如果需要回到框架原文，常量改 0
即等同 Z1。
